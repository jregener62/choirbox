Label Model erweitert

labels (Tabelle)
id int PK Primaerschluessel
name str(50) "Sopran", "Gitarre", "Schwierig"
color str(7) Hex-Farbe, z.B. "#ec4899"
category str(50)? "stimme" fuer Stimmen/Instrumente, null fuer allgemeine Labels
shortcode str(10)? NEU Kuerzel fuer Dateinamen: "S", "A", "Git", "PB". Wird fuer buildFilename() verwendet.
aliases str(200)? NEU Komma-getrennte Aliase fuer Fuzzy-Matching: "soprano,sop" fuer Sopran
choir_id FK Gehoert zu welchem Chor
sort_order int Reihenfolge in der Anzeige

Fuzzy Filename Parsing NEU

Automatische Erkennung beim Upload

Beim Upload wird der Dateiname gegen alle Stimmen-Labels (name + aliases) des Chors geprueft.

  • S-Africa-Refrain.mp3 → Sopran
  • sopran-africa.mp3 → Sopran
  • gitarre_intro.mp3 → Gitarre
  • playback-africa.mp3 → Playback
  • SA-Africa.mp3 → Sopran Alt
  • random-filename.mp3 → kein Label

Match-Strategie:
1. Exakter Shortcode-Match am Anfang: "S-..." → Sopran (wenn shortcode="S")
2. Name als Prefix/Substring: "sopran-..." → Sopran
3. Alias-Match: "soprano-..." → Sopran (wenn aliases="soprano,sop")
4. Kombinierte Shortcodes: "SA-..." → Sopran + Alt
Case-insensitive. Erster Treffer gewinnt.

Upload-Flow mit automatischem Labeling

1 User waehlt Datei(en) zum Upload
2 Import-Modal zeigt Dateiliste mit Vorschau
3 Fuzzy-Parse laeuft pro Datei: Stimmen-Labels des Chors werden gegen den Dateinamen geprueft
4 Erkannte Labels werden im Modal angezeigt (User kann aendern/entfernen vor Upload)
5 Upload + Label-Zuweisung — Datei wird hochgeladen, erkannte Labels automatisch gesetzt

Migrationsstrategie

Von hardcoded zu label-basiert
1 Seed-Labels erweitern: Default Stimmen-Labels (Sopran/S, Alt/A, Tenor/T, Bass/B) mit category="stimme" und shortcode beim ersten Start erstellen — wie bisher die allgemeinen Labels.
2 Label Model: Felder shortcode und aliases hinzufuegen (Migration).
3 Frontend: VoiceIcon liest Stimmen-Labels statt hardcoded VOICES. RenameModal zeigt Stimmen-Labels des Chors.
4 Fuzzy-Parse: Neuer Utility matchVoiceLabels(filename, labels[]) im Frontend.
